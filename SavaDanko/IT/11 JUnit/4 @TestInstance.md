Аннотация `@TestInstance` в JUnit позволяет управлять жизненным циклом тестового класса. Она определяет, создается ли новый экземпляр класса перед каждым тестом или используется один общий экземпляр на все тесты.

По умолчанию в JUnit каждый тестовый метод запускается на новом экземпляре тестового класса. Это гарантирует изоляцию тестов, но иногда создает неудобства, например, если нужно использовать общие ресурсы.

Аннотация `@TestInstance` позволяет изменить этот принцип и установить жизненный цикл `PER_CLASS`, при котором все тестовые методы запускаются в одном экземпляре класса.
```java
@TestInstance(TestInstance.Lifecycle.PER_CLASS)
public class MyTest {
    // Тестовые методы
}
```

| Параметр   | Описание                                              |
| ---------- | ----------------------------------------------------- |
| PER_METHOD | Новый экземпляр класса на каждый тест (по умолчанию). |
| PER_CLASS  | Один экземпляр класса на все тесты.                   |
