#linux 

---
`systemctl` — это основная команда для управления службами, процессами и состоянием системы на современных Linux-дистрибутивах, использующих систему инициализации **systemd**.

**systemd** — это система инициализации и менеджер системных служб. Она отвечает за запуск и управление демонами, монтирование файловых систем, настройку сети, точки монтирования и многое другое.

| Действие                  | Пример                             | Что делает                         |
| ------------------------- | ---------------------------------- | ---------------------------------- |
| Запуск службы             | `sudo systemctl start ssh`         | Запускает службу прямо сейчас      |
| Остановка службы          | `sudo systemctl stop apache2`      | Останавливает службу               |
| Перезапуск службы         | `sudo systemctl restart nginx`     | Перезапускает службу               |
| Проверка статуса          | `systemctl status ssh`             | Показывает подробный статус службы |
| Автозапуск при загрузке   | `sudo systemctl enable docker`     | Добавляет службу в автозагрузку    |
| Отключение автозапуска    | `sudo systemctl disable bluetooth` | Убирает службу из автозагрузки     |
| Перезагрузка всей системы | `sudo systemctl reboot`            | Перезагружает компьютер            |
| Завершение работы         | `sudo systemctl poweroff`          | Выключает компьютер                |

Вывести список всех известных unit-файлов:
```zsh
systemctl list-units --type=service
```

Перезапуск systemd после изменения unit-файлов:
```zsh
sudo systemctl daemon-reload
```
