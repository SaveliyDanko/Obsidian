###### **SSH-ключи**
SSH-ключи — это пара криптографических ключей:
- Приватный ключ (private key) — хранится у вас локально и должен быть защищён.
- Публичный ключ (public key) — размещается на сервере или сервисе, к которому вы хотите подключаться.
Они работают по принципу асимметричного шифрования, где публичный ключ используется для проверки, а приватный для подписи/дешифровки.

###### **Процесс логина на удалённый сервер по SSH**
Предположим, вы хотите подключиться к серверу по SSH:
1. Вы генерируете ключи:
    ```bash
    ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
    ```
    После этого у вас появляются:
    - `~/.ssh/id_rsa` — приватный ключ
    - `~/.ssh/id_rsa.pub` — публичный ключ
        
2. Вы копируете публичный ключ на сервер:
    ```bash
    ssh-copy-id user@remote_server_ip
    ```
    Или вручную добавляете содержимое `id_rsa.pub` в файл:
    ```
    ~/.ssh/authorized_keys
    ```
    на сервере от имени нужного пользователя.
    
3. При подключении:
    ```bash
    ssh user@remote_server_ip
    ```
    происходит следующее:
    - Клиент отправляет серверу информацию о поддерживаемых методах аутентификации.
    - Сервер предлагает аутентификацию по публичному ключу.
    - Клиент доказывает, что у него есть приватный ключ, соответствующий публичному.
    - Если проверка прошла успешно — доступ предоставляется.
**Важно:** Приватный ключ никогда не отправляется по сети.
