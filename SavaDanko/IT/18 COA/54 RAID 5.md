**RAID 5** — это развитие архитектуры RAID 4, в которой блоки данных и блоки четности (parity) распределяются по всем дискам массива циклически, устраняя выделенный диск для контрольных сумм.
- Минимальное количество дисков: 3.
- Доступная емкость: $(N - 1) \times S$, где $N$ — число дисков, $S$ — объем одного диска.
- Механизм избыточности: Контрольные суммы вычисляются через операцию XOR.
- Распределение четности: Блок четности для каждой полосы (stripe) смещается на следующий диск (например: в Stripe 1 четность на Диске 3, в Stripe 2 — на Диске 2, в Stripe 3 — на Диске 1).

###### **Причинно-следственные связи**
- Распределение блоков четности → Устранение «узкого места» (bottleneck). Операции записи распределяются по всем шпинделям массива, что позволяет выполнять несколько параллельных записей, если они не обращаются к одному и тому же физическому диску.
    
- Цикл «чтение-модификация-запись» → Сохранение Write Penalty. Каждое изменение блока по-прежнему требует 4 операций I/O (2 чтения и 2 записи), как и в RAID 4.
    
- Отказ одного диска → Переход в режим деградации (Degraded mode). Данные восстанавливаются на лету путем XOR-вычислений из оставшихся блоков и четности.
    
- Режим деградации → Значительное падение производительности. Каждое чтение с отсутствующего диска инициирует чтение со всех исправных дисков для реконструкции данных в оперативной памяти.
    
- Отказ второго диска → Полная и безвозвратная потеря данных. RAID 5 обладает отказоустойчивостью $N-1$.
