**RAID** — массив независимых дисков, воспринимаемый операционной системой как единый логический диск.
- Распределение данных: Блоки данных распределяются по всем физическим дискам массива.
- Избыточность: Использование части емкости для хранения контрольной информации с целью восстановления данных при отказах (характерно для большинства уровней, кроме RAID 0).
- Структура: Существует 7 уровней (0–6), которые не являются иерархическими, а определяют различные архитектуры.

###### **Причинно-следственные связи**
- Низкая скорость дисков относительно CPU/RAM → Необходимость параллельной работы нескольких накопителей.

- Распределение блока данных по дискам → Параллельное выполнение одного запроса ввода-вывода → Рост производительности.

- Наличие множества дисков → Параллельная обработка нескольких независимых запросов.

- Увеличение количества дисков → Повышение вероятности аппаратного сбоя → Необходимость избыточного кодирования.
    
- Замена плотных дисков массивом малых дисков → Рост эффективности I/O + возможность масштабирования емкости.
