**RAID 4** — это конфигурация массива, в которой данные разбиваются на блоки и распределяются по нескольким информационным дискам (как в RAID 0), но один диск в массиве выделяется исключительно для хранения контрольных сумм (четности).
- Минимальное количество дисков: 3.
- Доступная емкость: $(N - 1) \times S$, где $N$ — общее число дисков, а $S$ — объем наименьшего диска в массиве.
- Метод избыточности: Вычисление четности по алгоритму XOR (исключающее ИЛИ).
- Формула четности: $P = D1 \oplus D2 \oplus D3 \dots \oplus Dn$, где $D$ — блоки данных на соответствующих дисках.

###### **Причинно-следственные связи**
- Блочное чередование → Высокая скорость параллельного чтения. Каждый запрос на чтение может обрабатываться отдельным диском, что увеличивает общую пропускную способность.
    
- Любая запись данных → Необходимость обновления блока четности на выделенном диске.
    
- Использование одного диска для всех операций четности → «Узкое место» (bottleneck). При высокой интенсивности записи производительность всего массива ограничивается скоростью работы одного (выделенного) диска четности.
    
- Выход из строя любого информационного диска → Чтение в режиме деградации. Данные восстанавливаются «на лету» через операцию XOR между оставшимися дисками данных и диском четности.
