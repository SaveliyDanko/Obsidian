**Кольца привилегий**
- Ring 0 (Kernel mode): привилегированный код ОС.
- Ring 3 (User mode): пользовательские программы (ограниченные права).

**1) Монолитное ядро**
- Почти все подсистемы ОС (ФС, сеть, драйверы, процессы, память) работают внутри ядра (ring 0).
- Плюсы: высокая производительность (внутренние вызовы без IPC).
- Минусы: ошибка в драйвере/подсистеме может уронить всю ОС, большая поверхность атаки.
    
**2) Ядро с динамически загружаемыми модулями (модульный монолит)**
- То же, что монолит, но функциональность добавляется модулями, которые можно загружать/выгружать во время работы (тоже ring 0).
- Плюсы: гибкость + почти монолитная скорость.
- Минусы: риск остаётся (модуль в ядре может “положить” систему), важны контроль/подписи модулей.

**3) Микроядро**
- В ядре (ring 0) оставляют минимум (планировщик, базовая память, IPC (Inter-Process Communication)).
- Остальное (ФС, сеть, драйверы) — службы в user space (ring 3), общение через IPC.
- Плюсы: выше надёжность и безопасность (меньше кода в ring 0, сервисы можно перезапускать).
- Минусы: больше накладных расходов (IPC, переключения контекста), сложнее добиться скорости монолита.
