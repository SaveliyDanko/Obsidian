###### **Суть принципа**
Принцип KISS гласит:
Система или код должны быть настолько простыми, насколько это возможно.  
###### **Пиши понятный, а не "умный" код**
Плохо:
```java
// Одной строкой, но тяжело читается
return (x & 1) == 0 ? "even" : "odd";
```

Хорошо:
```java
if (x % 2 == 0) {
    return "even";
} else {
    return "odd";
}
```

###### **Избегай ненужных абстракций**
Не стоит сразу делать сложную архитектуру, если задача простая:
```java
// Не нужно 5 интерфейсов и фабрик для простого логгера
Logger logger = new SimpleConsoleLogger();
```

###### **Разделяй большие функции**
Плохо:
```java
void process() {
    // 200 строк в одной функции
}
```

Хорошо:
```java
void process() {
    loadData();
    validate();
    transform();
    save();
}
```

###### **Следи за читаемостью**
- Названия переменных и функций должны быть говорящими.
- Лучше написать на 3 строки больше, но чтобы было понятно.

###### **KISS ≠ Примитивно**
Простота ≠ примитив. Это умная простота, которая решает задачу без перегруза.
