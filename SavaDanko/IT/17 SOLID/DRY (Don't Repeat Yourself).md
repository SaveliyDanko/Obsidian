###### **Почему важно избегать повторений?**
Если одинаковый код находится в нескольких местах:
- Изменения становятся сложными: нужно править каждую копию вручную.    
- Ошибки труднее отследить и исправить.
- Поддержка кода становится дороже.
- Чтение и понимание кода усложняется.
    
###### **Выносить повторяющийся код в функции или методы**
```java
// Повторение (плохо)
int area1 = width1 * height1;
int area2 = width2 * height2;

// DRY-версия (хорошо)
int calcArea(int width, int height) {
    return width * height;
}
```

###### **Создание абстракций**
Если шаблон повторяется на уровне поведения или структуры — стоит подумать о:
- Классе / интерфейсе / обобщении
- Шаблоне проектирования (например, Strategy, Template Method и др.)

###### **Использование констант и конфигураций**
```python
# Плохо
print("Ошибка подключения к серверу")
log("Ошибка подключения к серверу")

# Хорошо
ERROR_MSG = "Ошибка подключения к серверу"
print(ERROR_MSG)
log(ERROR_MSG)
```

###### **DRY на уровне SQL / бизнес-логики / документации**
DRY применим не только к коду, но и к:
- SQL-запросам
- бизнес-правилам
- сообщениям об ошибках
- документации
