`@Bean` — это аннотация, с помощью которой ты описываешь компонент, который должен быть создан, управляем и храниться в контейнере Spring (в `ApplicationContext`).

###### **Где используется**
Обычно `@Bean` применяется внутри классов, помеченных как:
```java
@Configuration
public class AppConfig {
    
    @Bean
    public MyService myService() {
        return new MyServiceImpl();
    }
}
```
Здесь:
- `@Configuration` говорит, что класс содержит конфигурацию Spring
- метод, помеченный `@Bean`, возвращает объект который будет зарегистрирован в контейнере под именем `myService`.

###### **Что делает Spring**
- При старте приложения Spring сканирует конфигурации.  
- Находит методы с `@Bean`.  
- Вызывает эти методы и сохраняет возвращённые объекты в ApplicationContext.
- Потом эти объекты можно внедрять (инжектить) через `@Autowired`, `@Inject`, `@Resource` и т. д.
