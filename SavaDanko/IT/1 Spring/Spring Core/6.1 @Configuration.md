`@Configuration` — это аннотация, которая помечает класс как источник конфигурации Spring.
Внутри такого класса можно определять бины при помощи `@Bean`.

###### **Пример**
```java
@Configuration
public class AppConfig {

    @Bean
    public MyService myService() {
        return new MyServiceImpl();
    }
}
```

###### **Что делает Spring под капотом**
- При запуске Spring создаёт экземпляр класса, помеченного `@Configuration`
- Сканирует методы с `@Bean`
- Вызывает эти методы один раз и помещает возвращённые объекты в `ApplicationContext`
- Все зависимости между бинами автоматически связываются (через DI — dependency injection)
