`@Around` — это (advice), который оборачивает вызов метода целиком. Ты получаешь возможность:
- выполнить код **до вызова метода**,
- **вызвать метод вручную** (или не вызывать вообще),
- **изменить возвращаемое значение**,
- **перехватить исключение**.

```java
@Around("pointcut-expression")
public Object aroundAdvice(ProceedingJoinPoint pjp) throws Throwable {
    // ...
}
```
- `ProceedingJoinPoint` — позволяет выполнить метод через `proceed()`.
- Метод всегда должен возвращать `Object`.
- Обязательно вызывать `pjp.proceed()`, если ты хочешь, чтобы оригинальный метод выполнился.
