Это **мета-аннотация конфигурации**, которая **включает обработку аспектов** (`@Aspect`) в Spring-приложении.

```java
@Configuration
@EnableAspectJAutoProxy
public class AppConfig {
    // конфигурация бинов
}
```

#### Зачем она нужна?
Без `@EnableAspectJAutoProxy`, даже если ты напишешь `@Aspect`, Spring **не будет обрабатывать аспекты** — то есть:
- `@Before`, `@After`, `@Around` и другие методы не сработают.
- Аспекты просто проигнорируются.
> Она активирует механизм **"авто-проксификации"**, который оборачивает нужные объекты в прокси, способные перехватывать вызовы методов.

#### Как работает под капотом?
Когда Spring видит `@EnableAspectJAutoProxy`:
1. Он включает `AnnotationAwareAspectJAutoProxyCreator` в контекст.
2. Этот компонент:
    - ищет все классы с `@Aspect`,
    - создаёт прокси-объекты для бинов, к которым применим pointcut,
    - перехватывает вызовы методов и применяет `@Before`, `@After`, `@Around` и т.д.
