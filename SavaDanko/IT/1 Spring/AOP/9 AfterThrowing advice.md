`@AfterThrowing` — это advice, который срабатывает после того, как метод выбросил исключение.
То есть, если метод завершился с ошибкой, Spring «впрыгивает» в момент выброса исключения, и выполняет твой код — например, логирует, отправляет письмо, пишет в базу и т.д.

`@AfterThrowing` не перехватывает исключение, оно всё равно пробрасывается вверх.
Это только уведомление: «исключение произошло — вот тебе `Throwable` — делай, что хочешь».

```java
@AfterThrowing(
    pointcut = "execution(...)", 
    throwing = "ex"
)
public void handleException(JoinPoint joinPoint, Throwable ex) {
    // обработка ошибки
}
```

`throwing` - имя параметра метода, в который будет передано исключение
`Throwable ex` - само исключение, которое было выброшено
