`JSESSIONID` — это уникальный идентификатор HTTP-сессии, который сервер (например, Tomcat) создаёт для каждого пользователя при первом обращении к приложению.

Он нужен для того, чтобы сервер мог “узнавать” пользователя между запросами, так как HTTP — протокол без состояния (каждый запрос независим).

###### **Как это работает**
Когда пользователь впервые обращается к приложению, сервер создаёт новую сессию:
```java
HttpSession session = request.getSession(true);
```

Сервер присваивает этой сессии уникальный идентификатор, например:
```
JSESSIONID = 5F7E4B3D2C1A09E12345ABCDEF
```

Этот ID отправляется клиенту (браузеру) в HTTP-cookie:
```
Set-Cookie: JSESSIONID=5F7E4B3D2C1A09E12345ABCDEF; Path=/; HttpOnly
```

При каждом последующем запросе браузер автоматически добавляет этот cookie:
```
Cookie: JSESSIONID=5F7E4B3D2C1A09E12345ABCDEF
```

Сервер по значению `JSESSIONID` находит соответствующий объект сессии в памяти и восстанавливает данные, например:
- данные авторизованного пользователя (SecurityContext)
- параметры корзины в интернет-магазине
- другие пользовательские атрибуты
