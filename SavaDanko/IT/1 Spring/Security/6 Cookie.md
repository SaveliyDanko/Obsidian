**Cookie** — это небольшой фрагмент данных, который веб-сервер сохраняет в браузере пользователя. При каждом последующем запросе к этому же серверу браузер автоматически отправляет эти данные обратно.

Проще говоря, cookie — это “память” сайта о пользователе.
С их помощью сайт “узнаёт” тебя между запросами, даже несмотря на то, что HTTP сам по себе не сохраняет состояние.

###### **Как это работает**
Клиент (браузер) отправляет запрос:
```
GET /login HTTP/1.1
Host: example.com
```

Сервер отвечает и добавляет в заголовок:
```
Set-Cookie: userId=123; Path=/; HttpOnly
```

Теперь браузер сохраняет cookie с именем `userId` и значением `123`.

При следующих запросах браузер добавляет cookie в заголовок:
```
Cookie: userId=123
```

Сервер, получив это, понимает, какой именно пользователь сделал запрос.

###### **Что хранится в Cookie**
Cookie — это пары `ключ = значение`, например:
```text
JSESSIONID=5F7E4B3D2C1A09E12345ABCDEF
user_theme=dark
lang=ru
```
