Open System Interconnection Model - это эталонная сетевая модель для стандартизации взаимодействия различных сетевых систем.
Она разбивает процесс сетевого взаимодействия на 7 уровней:

###### 1. Физический уровень (Physical Layer)
- Отвечает за: передачу необработанных битов по физическому каналу связи.
- Примеры: кабели, радиоволны.

###### 2. Канальный уровень (Data Link Layer)
- Отвечает за: установку, поддержание и завершение соединения между двумя узлами в одной сети. Также исправляет ошибки, возникающие на физическом уровне.
- Примеры протоколов: Ethernet.

###### 3. Сетевой уровень (Network Layer)
- Отвечает за: маршрутизацию пакетов между разными сетями.
- Примеры протоколов: IP (IPv4, IPv6).
- Устройства: маршрутизаторы (routers).

###### 4. Транспортный уровень (Transport Layer)
- Отвечает за: доставку данных от процесса к процессу, контроль потока, надёжность.
- Протоколы: TCP (надёжный), UDP (ненадёжный, но быстрый).
- Функции: разбиение на сегменты, повторная сборка, контроль потерь.

###### 5. Сеансовый уровень (Session Layer)
- Отвечает за: установку, поддержку и завершение сеансов между приложениями.
- Функции: управление диалогом (кто, когда говорит), восстановление после сбоев.

###### 6. Представительский уровень (Presentation Layer)
- Отвечает за: представление данных в нужном формате — кодировка, шифрование, сжатие.
- Примеры: JPEG, MPEG, SSL, ASCII.

###### 7. Прикладной уровень (Application Layer)
- Отвечает за: интерфейс между приложением и сетью. Пользователь взаимодействует именно с этим уровнем.
- Примеры: HTTP, FTP, DNS.
