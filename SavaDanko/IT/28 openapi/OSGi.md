**OSGi (Open Services Gateway initiative)** — это спецификация для модульной системы и динамической платформы Java, которая позволяет:
- разбивать приложение на независимые модули    
- загружать, обновлять и удалять их на лету (без перезапуска JVM)
- управлять зависимостями и версиями модулей
Каждый модуль в OSGi называется bundle (по сути, это JAR с дополнительными метаданными в `MANIFEST.MF`).

###### **Зачем нужен OSGi**
В обычном Java-приложении:
- все зависимости загружаются в один classpath
- возникает «ад dependency hell» (конфликты версий библиотек)
- для обновлений часто приходится перезапускать приложение

OSGi решает эти проблемы:
- изоляция классов по модулям
- разные версии библиотек могут жить одновременно
- обновления модулей «на горячую»
- явное описание зависимостей
