В Java интерфейс `Lock` из пакета `java.util.concurrent.locks` представляет собой расширенную альтернативу ключевому слову `synchronized`, предоставляя более гибкий и мощный механизм управления многопоточностью.
    
| Класс                    | Описание                                                  |
| ------------------------ | --------------------------------------------------------- |
| `ReentrantLock`          | Стандартная рекурсивная блокировка, аналог `synchronized` |
| `ReentrantReadWriteLock` | Блокировка с разделением прав на чтение и запись          |
| `StampedLock` (Java 8+)  | Современная блокировка с оптимистичным чтением            |
    
- Поддерживает методы:
    - `lock()` — блокировка, если занята, поток ждёт;
    - `tryLock()` — неблокирующая попытка захвата;        
    - `tryLock(long time, TimeUnit unit)` — попытка захвата с таймаутом;
    - `lockInterruptibly()` — захват с возможностью прерывания;
    - `isLocked()` — проверка, заблокирован ли объект;
    - `isHeldByCurrentThread()` — держит ли текущий поток блокировку.
