Пакет `java.util.concurrent.atomic` в Java предоставляет набор атомарных классов, которые позволяют безопасно изменять переменные из нескольких потоков без использования `synchronized` или других явных блокировок.

Эти классы используют низкоуровневые атомарные операции процессора, такие как **CAS (Compare-And-Swap)**, чтобы избежать гонок данных.

#### Основные классы `java.util.concurrent.atomic`

| Класс                     | Назначение                     |
| ------------------------- | ------------------------------ |
| `AtomicInteger`           | Атомарное целое число (`int`)  |
| `AtomicLong`              | Атомарное целое число (`long`) |
| `AtomicBoolean`           | Атомарный булевый флаг         |
| `AtomicReference<V>`      | Атомарная ссылка на объект     |
| `AtomicIntegerArray`      | Атомарный массив целых чисел   |
| `AtomicLongArray`         | Атомарный массив `long`        |
| `AtomicReferenceArray<V>` | Атомарный массив ссылок        |

#### Внутренности: что такое CAS?
CAS (Compare-And-Swap) — низкоуровневая операция:
1. Сравнивает текущее значение переменной с ожидаемым.
2. Если они равны — записывает новое значение.    
3. Если нет — операция не выполняется, можно повторить.
Это позволяет избежать блокировок и гонок.
