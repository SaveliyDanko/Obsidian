Ссылка — это альтернативное имя (или псевдоним) для уже существующей переменной. Она не копирует значение, а просто указывает на ту же память.

Простое определение:
```cpp
int a = 10;
int& ref = a; // ref — это ссылка на a
```
Теперь `ref` и `a` — одно и то же значение. Изменение через `ref` влияет на `a`, и наоборот.

#### <mark style="background: #ABF7F7A6;">Основные свойства ссылок</mark>
###### 1. **Ссылка должна быть инициализирована при создании**:
```cpp
int a;
int& r; // ❌ ошибка — нужно сразу инициализировать
```

###### 2. **Ссылку нельзя переназначить**:
```cpp
int a = 10, b = 20;
int& r = a;
r = b;    // ❗ r теперь содержит значение b, но всё ещё ссылается на a
```
 
###### 3. **Работают как синоним**: ссылка и переменная — это один и тот же объект.

###### 4. **Не существует "null-ссылок"** (в отличие от указателей).

#### <mark style="background: #ABF7F7A6;">Ссылки и функции</mark>
Ссылки часто используют при передаче аргументов в функции, чтобы избежать копирования или позволить функции изменить аргумент.

#### Передача по ссылке:
```cpp
void increment(int& x) {
    x++;
}

int main() {
    int a = 5;
    increment(a); // a теперь 6
}
```

#### <mark style="background: #ABF7F7A6;">Ссылки как const</mark>
Можно объявить константную ссылку:
```cpp
void print(const int& x) {
    cout << x;
}
```

**Преимущества:**
- можно передавать большие объекты (например, строки или векторы) без копирования, но при этом гарантировать, что функция их не изменит.

#### <mark style="background: #ABF7F7A6;">Ссылки и возвращаемые значения</mark>
Можно возвращать ссылку из функции, если переменная живёт за пределами функции (например, глобальная переменная или элемент массива/вектора):
```cpp
int& getElement(int arr[], int index) {
    return arr[index];
}
```
