**Функциональное программирование (ФП)** — это парадигма, основанная на использовании чистых функций. Оно относится к декларативному стилю, сосредоточенному на «что решать», в отличие от императивного стиля, ориентированного на «как решать». В ФП используются выражения (возвращающие значения), а не операторы (изменяющие состояние).

ФП основывается на лямбда-исчислении

#### <mark style="background: #ABF7F7A6;">Концепции функционального программирования</mark>
##### ==Чистые функции==
Чистые функции обладают двумя ключевыми свойствами:
1. **Детерминированность** – при одинаковых входных данных всегда возвращают одинаковый результат.
2. **Отсутствие побочных эффектов** – не изменяют аргументы, переменные, файлы или потоки ввода/вывода (**неизменяемость**).

##### ==Рекурсия==
В функциональных языках **нет циклов**. Вместо них используется **рекурсия**.

##### ==Функции первого класса и функции высшего порядка==
Функции в функциональном программировании являются **объектами первого класса** – их можно передавать в качестве аргументов, возвращать из других функций и хранить в структурах данных.

Функции высшего порядка принимают другие функции как аргументы или возвращают их.

**Пример:**
```js
function show_output(f) {
    f(); // вызов переданной функции
}

function print_gfg() {
    console.log("hello gfg");
}

show_output(print_gfg); // передача функции как аргумента
```

#### ==Почему функциональное программирование важно?==
Функциональное программирование имеет ряд преимуществ, соответствующих современным требованиям к разработке программного обеспечения:
- **Конкурентность и параллелизм** – функции без побочных эффектов безопасны для параллельного выполнения, так как не изменяют общие переменные.
- **Читаемость и сопровождаемость кода** – небольшие детерминированные функции, характерные для функциональных языков, упрощают написание, тестирование и отладку кода.
- **Ленивая оценка** – вычисления выполняются только при необходимости, что может повысить производительность и избежать лишних вычислений.
