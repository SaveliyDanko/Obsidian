#### <mark style="background: #ABF7F7A6;">Основные сегменты памяти в программе на C++:</mark>
#### ==1. Code segment (Text segment)==
- **Что хранит:**  
    Исполняемый машинный код программы (компилированные инструкции).
- **Особенности:**
    - Обычно является доступным только для чтения (`read-only`), чтобы предотвратить случайные изменения инструкций.
    - Хранится непосредственно в исполняемом файле программы.

#### ==2. Data segment==
#### Делится на две части:
##### a. **Инициализированный сегмент данных (Initialized Data Segment)**
- **Что хранит:**  Глобальные и статические переменные, которые явно инициализированы программистом.

- **Пример:**
```cpp
int global_var = 10;
static int static_var = 20;
```
- **Особенности:** Значения этих переменных определяются во время компиляции и сохраняются в исполняемом файле.

#### b. **Неинициализированный сегмент данных (BSS Segment)**
- **Что хранит:**  Глобальные и статические переменные, которые не инициализированы явно (или инициализированы нулём).

- **Пример:**
```cpp
int global_uninitialized_var;
static int static_uninitialized_var;
```
- **Особенности:** Переменные автоматически инициализируются нулём на этапе запуска программы, однако не хранятся в самом исполняемом файле, что позволяет экономить место.

#### ==4. Heap (Куча, динамическая память)==
- **Что хранит:** Динамически выделенные объекты (используя `new`, `malloc` и т.д.).

- **Пример:**
```cpp
int* p = new int(42); // выделение памяти в куче
```
 
- **Особенности:**
    - Требует явного выделения и освобождения памяти программистом (`new`/`delete`).
    - Размер кучи обычно намного больше стека.
    - Память выделяется и освобождается в произвольном порядке.
