Интеграция базы данных — это процесс объединения данных из разных источников (чаще всего — из разных баз данных) в единую, согласованную и доступную для использования систему. Это позволяет организациям работать с данными так, как будто они находятся в одном месте, даже если на самом деле они распределены по разным системам.

###### **Основные цели интеграции:**
1. Объединить данные — собрать данные из разных источников в одном месте. 
2. Согласовать данные — привести их к общему формату (например, одинаковое написание дат, валют, имён).
3. Упростить доступ — дать возможность работать с данными без необходимости знать, где и как они хранятся.
4. Поддерживать актуальность — следить за тем, чтобы изменения в одном источнике отражались в других.

###### **Виды интеграции:**

| Вид                    | Краткое описание                                                                                                                |
| ---------------------- | ------------------------------------------------------------------------------------------------------------------------------- |
| Физическая интеграция  | Перенос всех данных в одну центральную БД (например, слияние нескольких таблиц в одну).                                         |
| Виртуальная интеграция | Создание единого интерфейса доступа, при этом сами данные остаются в исходных БД (например, через API или виртуальные таблицы). |
| ETL-интеграция         | Классический подход: данные Extract (извлекаются), Transform (преобразуются) и Load (загружаются) в хранилище данных.           |
