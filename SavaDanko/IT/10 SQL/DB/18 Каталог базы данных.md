**Каталог** (системный каталог, data dictionary, system catalog) — это встроенная служебная структура, которая содержит метаданные: информацию о самих базах данных, таблицах, индексах, пользователях, правах доступа и других объектах.


###### **Что хранится в каталоге?**

| Объект                 | Что описывается                                                      |
| ---------------------- | -------------------------------------------------------------------- |
| Базы данных, схемы     | Названия, владельцы, параметры                                       |
| Таблицы и их столбцы   | Имена, типы, порядки, ограничения (`NOT NULL`, `DEFAULT`, и т.д.)    |
| Индексы                | Поля, типы индекса, уникальность                                     |
| Ограничения            | `PRIMARY KEY`, `UNIQUE`, `FOREIGN KEY`, `CHECK`                      |
| Представления (`VIEW`) | Определения (хранимые SQL-запросы)                                   |
| Пользователи и роли    | Пароли, права доступа, роли, принадлежность                          |
| Функции и процедуры    | Имя, типы аргументов, возвращаемые значения, язык (`plpgsql`, `sql`) |
| Статистика             | Кардинальность, селективность, распределения значений                |

###### **Каталог в PostgreSQL**
PostgreSQL хранит системный каталог в pg_catalog — это встроенная схема, доступная как обычные таблицы:

|Таблица|Описание|
|---|---|
|`pg_class`|Таблицы и индексы|
|`pg_attribute`|Столбцы таблиц|
|`pg_namespace`|Схемы|
|`pg_type`|Типы данных|
|`pg_index`|Индексы|
|`pg_roles`|Роли|
|`pg_proc`|Функции и процедуры|
|`pg_constraint`|Ограничения|

Также есть стандартизированная зона `information_schema`, поддерживаемая многими СУБД.

###### **Пример:**
Хочешь узнать, какие таблицы есть в текущей базе PostgreSQL?
```sql
SELECT tablename FROM pg_catalog.pg_tables WHERE schemaname = 'public';
```

Или посмотреть все столбцы таблицы:
```sql
SELECT column_name, data_type
FROM information_schema.columns
WHERE table_name = 'users';
```
