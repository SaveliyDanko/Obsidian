В базах данных схема — это логическая структура, которая объединяет связанные между собой объекты: таблицы, представления, индексы, функции, процедуры, типы и т. д.

Схема действует как контейнер или пространство имён, позволяющее организовать объекты внутри базы данных.

Схема базы данных — это логическая единица группировки объектов, позволяющая:
- структурировать содержимое БД,
- управлять правами доступа,
- избежать конфликтов имён.
    
Пример (в PostgreSQL):
```sql
CREATE SCHEMA accounting;

CREATE TABLE accounting.invoices (
  id SERIAL PRIMARY KEY,
  amount NUMERIC,
  issued_at DATE
);
```
- `accounting` — это схема.
- `invoices` — таблица внутри схемы `accounting`.
- Полное имя: `accounting.invoices`.
    

###### **Как работает схема**
В СУБД схема:
- Является вложенной в базу данных.    
- Может иметь своего владельца (роль/пользователь).
- Имеет пространство имён: в одной базе могут быть схемы с одноимёнными таблицами, но в разных схемах.
```sql
-- В схеме public
CREATE TABLE public.users (id INT, name TEXT);

-- В схеме crm
CREATE TABLE crm.users (id INT, email TEXT);
```
Теперь в одной базе могут быть две таблицы `users`, но в разных схемах.
