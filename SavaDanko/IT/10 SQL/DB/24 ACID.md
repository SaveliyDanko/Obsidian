**ACID** — это набор свойств, которые гарантируют корректное выполнение транзакций в базе данных.

Транзакция — это группа операций, которая выполняется как единое целое.

**ACID = Atomicity + Consistency + Isolation + Durability**

###### **A — Atomicity (Атомарность)**
Транзакция выполняется полностью, или не выполняется вообще.

###### **C — Consistency (Согласованность)**
После завершения транзакции данные должны находиться в корректном состоянии, соответствовать правилам:
- ограничениям целостности
- CHECK-ограничениям
- бизнес-правилам

###### **I — Isolation (Изоляция)**
Транзакции не должны мешать друг другу.
Каждая должна работать так, как будто она единственная.

Уровни изоляции:

| Уровень          | Гарантии                                                       |
| ---------------- | -------------------------------------------------------------- |
| READ UNCOMMITTED | можно читать чужие незаконченые изменения → грязные чтения     |
| READ COMMITTED   | грязных чтений нет, но данные могут меняться между запросами   |
| REPEATABLE READ  | данные повторяемы, но возможны фантомы                         |
| SERIALIZABLE     | транзакции работают так, как будто выполняются последовательно |

###### **D — Durability (Надёжность / Сохранность)**
Если транзакция зафиксирована (COMMIT), то её результат будет сохранён, даже если:
- выключили сервер
- оборвали питание
- произошла ошибка ядра
- упал процесс PostgreSQL или MySQL
