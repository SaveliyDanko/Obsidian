В PostgreSQL операторы `ANY` и `ALL` используются в сочетании с подзапросами или массивами, чтобы сравнить значение с набором значений. Это мощные инструменты для гибкого сравнения: "хотя бы с одним" (`ANY`) или "со всеми" (`ALL`).

###### **`ANY` — «хотя бы один»**
Сравнивает выражение с каждым элементом множества, и возвращает `TRUE`, если хотя бы одно сравнение истинно.
```sql
x оператор ANY (множество)
```

###### **С подзапросом:**
```sql
SELECT * FROM orders
WHERE total > ANY (
  SELECT total FROM orders WHERE customer_id = 42
);
```
Найдёт заказы, у которых сумма больше хотя бы одного заказа клиента №42.

###### **С массивом:**
```sql
SELECT * FROM products
WHERE category = ANY (ARRAY['Books', 'Electronics']);
```
То же, что и `IN ('Books', 'Electronics')`.

###### **`ALL` — «все элементы удовлетворяют»**
Сравнивает выражение с каждым элементом множества, и возвращает `TRUE`, только если все сравнения истинны.
```sql
x оператор ALL (множество)
```

###### **С подзапросом:**
```sql
SELECT * FROM orders
WHERE total > ALL (
  SELECT total FROM orders WHERE customer_id = 42
);
```
Найдёт заказы, у которых сумма больше всех заказов клиента №42 (т.е. максимума).

###### **С массивом:**
```sql
SELECT * FROM reviews
WHERE rating >= ALL (ARRAY[4, 5]);
```
Найдёт отзывы с рейтингом не ниже, чем 4 и 5 → то есть только 5.
