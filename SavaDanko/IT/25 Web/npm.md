**npm** (Node Package Manager) — это менеджер пакетов для Node.js.

Когда ты создаешь проект на Node.js, ты обычно начинаешь с команды:
```bash
npm init
```
Она создаёт файл `package.json`, где будут храниться все метаданные проекта — список зависимостей, скрипты и прочее.

###### `package.json`
```json
{
  "name": "my-project",
  "version": "1.0.0",
  "scripts": {
    "start": "node index.js",
    "dev": "nodemon index.js"
  },
  "dependencies": {
    "express": "^4.18.0"
  },
  "devDependencies": {
    "nodemon": "^2.0.22"
  }
}
```
- `dependencies` — библиотеки, нужные в продакшене
- `devDependencies` — только для разработки (например, линтеры, тесты)
    
| Команда                  | Описание                                      |
| ------------------------ | --------------------------------------------- |
| `npm install <имя>`      | Установка пакета                              |
| `npm install`            | Установка всех зависимостей из `package.json` |
| `npm install --save`     | (по умолчанию) добавляет в `dependencies`     |
| `npm install --save-dev` | Добавляет в `devDependencies`                 |
| `npm uninstall <имя>`    | Удаление пакета                               |

###### Структура после установки
После установки пакеты попадают в папку:
```
node_modules/
```
а их список и версии — в файл `package-lock.json`.

---

###### Полезные команды

| Команда        | Назначение                                    |
| -------------- | --------------------------------------------- |
| `npm start`    | Выполнить скрипт `start` из `package.json`    |
| `npm run dev`  | Любой кастомный скрипт (`dev` в примере выше) |
| `npm outdated` | Показать устаревшие зависимости               |
| `npm update`   | Обновить зависимости                          |
| `npm list`     | Показать установленные пакеты                 |
| `npx <пакет>`  | Запустить пакет без установки                 |
