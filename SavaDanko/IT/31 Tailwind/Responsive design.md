Каждый утилитарный класс в Tailwind может применяться условно — на разных брейкпоинтах (точках останова), что делает построение сложных адаптивных интерфейсов предельно простым, без необходимости выходить за рамки HTML.

Сначала убедитесь, что в разделе `<head>` вашего документа добавлен метатег viewport:
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0" />
```

Чтобы применить утилиту только на определённом брейкпоинте, достаточно добавить к имени утилиты префикс брейкпоинта, за которым следует двоеточие:
```html
<!-- Ширина 16 по умолчанию, 32 на средних экранах и 48 на больших -->
<img class="w-16 md:w-32 lg:w-48" src="..." />
```

По умолчанию Tailwind использует пять брейкпоинтов, основанных на типичных разрешениях устройств:

| Префикс брейкпоинта | Минимальная ширина | CSS-условие                       |
| ------------------- | ------------------ | --------------------------------- |
| `sm`                | 40rem (640px)      | `@media (width >= 40rem) { ... }` |
| `md`                | 48rem (768px)      | `@media (width >= 48rem) { ... }` |
| `lg`                | 64rem (1024px)     | `@media (width >= 64rem) { ... }` |
| `xl`                | 80rem (1280px)     | `@media (width >= 80rem) { ... }` |
| `2xl`               | 96rem (1536px)     | `@media (width >= 96rem) { ... }` |

###### **Мобильный подход (Mobile-first)**
Tailwind использует mobile-first систему брейкпоинтов, аналогичную той, что применяется в других фреймворках, например в Bootstrap.

Это означает, что утилиты без префиксов (например, `uppercase`) применяются ко всем размерам экранов, а утилиты с префиксами (например, `md:uppercase`) начинают действовать только на указанном брейкпоинте и выше.

###### **Стилизация для мобильных устройств**
Главная особенность такого подхода — чтобы задать стиль для мобильных экранов, нужно использовать утилиту без префикса.

Поэтому в Tailwind рекомендуется сначала реализовать макет для мобильных устройств, а затем постепенно добавлять изменения для более крупных экранов — сначала для `sm`, затем `md`, `lg` и так далее.

###### **Нацеливание на диапазон брейкпоинтов**
По умолчанию стили, применяемые через классы вроде `md:flex`, начинают действовать с указанного брейкпоинта и продолжают действовать на всех более крупных экранах.

Если вы хотите, чтобы утилита применялась только в пределах определённого диапазона брейкпоинтов, вы можете объединить responsive-вариант (например, `md`) с вариантом `max-*`, чтобы ограничить действие стиля нужным диапазоном:
```html
<div class="md:max-xl:flex">
  <!-- ... -->
</div>
```
Tailwind автоматически генерирует вариант `max-*` для каждого брейкпоинта.  
Таким образом, «из коробки» доступны следующие варианты:

| Вариант   | Медиа-запрос                     |
| --------- | -------------------------------- |
| `max-sm`  | `@media (width < 40rem) { ... }` |
| `max-md`  | `@media (width < 48rem) { ... }` |
| `max-lg`  | `@media (width < 64rem) { ... }` |
| `max-xl`  | `@media (width < 80rem) { ... }` |
| `max-2xl` | `@media (width < 96rem) { ... }` |

###### **Настройка темы**
Вы можете настроить свои брейкпоинты, используя тематические переменные `--breakpoint-*`:
```css
@theme {
  --breakpoint-xs: 30rem;
  --breakpoint-2xl: 100rem;
  --breakpoint-3xl: 120rem;
}
```

###### **Использование произвольных значений (Arbitrary values)**
Если вам нужно применить уникальный, разовый брейкпоинт, который не имеет смысла добавлять в общую тему проекта, можно использовать варианты `min` или `max`, чтобы создать кастомный брейкпоинт “на лету” с любым произвольным значением:
```html
<div class="max-[600px]:bg-sky-300 min-[320px]:text-center">
  <!-- ... -->
</div>
```

###### **Container queries**
Container queries (запросы к контейнеру) — это современная возможность CSS, позволяющая изменять стили элемента в зависимости от размеров его родительского контейнера, а не от размера всего окна браузера.

Чтобы использовать container queries в Tailwind, отметьте элемент как контейнер с помощью класса `@container`, а затем применяйте варианты вроде `@sm` или `@md` к дочерним элементам, чтобы менять их стили в зависимости от ширины контейнера:
```html
<div class="@container">
  <div class="flex flex-col @md:flex-row">
    <!-- ... -->
  </div>
</div>
```

###### **Именованные контейнеры (Named containers)**
В более сложных макетах с несколькими вложенными контейнерами можно присваивать имена с помощью синтаксиса `@container/{name}` и затем настраивать стили, ориентируясь на конкретный контейнер, а не только на ближайший:
```html
<div class="@container/main">
  <!-- ... -->
  <div class="flex flex-row @sm/main:flex-col">
    <!-- ... -->
  </div>
</div>
```
Такой подход позволяет применять стили в зависимости от размера определённого (в том числе удалённого) контейнера, а не только ближайшего родителя.

###### **Настройка пользовательских размеров контейнера (Custom container sizes)**
Вы можете задать свои размеры контейнера, используя тематические переменные `--container-*`:
```css
@theme {
  --container-8xl: 96rem;
}
```

Это добавит новый вариант container query `@8xl`, который можно использовать в разметке:
```html
<div class="@container">
  <div class="flex flex-col @8xl:flex-row">
    <!-- ... -->
  </div>
</div>
```

###### **Использование произвольных значений (Arbitrary container query values)**
Если вам нужно создать разовый запрос к контейнеру с произвольным размером, используйте варианты вроде `@min-[475px]` или `@max-[960px]`:
```html
<div class="@container">
  <div class="flex flex-col @min-[475px]:flex-row">
    <!-- ... -->
  </div>
</div>
```

###### **Использование единиц измерения container query (Container query units)**
Вы можете использовать специальные единицы измерения контейнера (например, `cqw`) в других утилитах Tailwind, чтобы привязать размеры к ширине контейнера:
```html
<div class="@container">
  <div class="w-[50cqw]">
    <!-- ... -->
  </div>
</div>
```
В этом примере элемент занимает 50% ширины контейнера, независимо от ширины окна браузера.
