**Playbook** — это автоматизационный чертёж в формате YAML, который Ansible использует для развертывания и настройки управляемых узлов.
- Playbook  
    Набор plays, который определяет порядок выполнения операций Ansible сверху вниз для достижения общей цели.
    
- Play  
    Упорядоченный список задач, сопоставляемых с управляемыми узлами из inventory.
    
- Task (задача)  
    Ссылка на отдельный модуль, который определяет выполняемую Ansible операцию.
    
- Module (модуль)  
    Единица кода или бинарный объект, запускаемый Ansible на управляемых узлах.
    
    Модули Ansible сгруппированы в коллекции и имеют полностью квалифицированное имя (FQCN, Fully Qualified Collection Name).

---
###### Выполните следующие шаги, чтобы создать playbook, который пингует ваши хосты и выводит сообщение «Hello world»:
1. Создайте файл с именем `playbook.yaml` в каталоге `ansible_quickstart`, который вы создали ранее, со следующим содержимым:
```yaml
- name: My first play
  hosts: myhosts
  tasks:
    - name: Ping my hosts
      ansible.builtin.ping:

    - name: Print message
      ansible.builtin.debug:
        msg: Hello world
```

2. Запустите ваш playbook:
```bash
ansible-playbook -i inventory.ini playbook.yaml
```

3. Ansible вернёт следующий вывод:
```plaintext
PLAY [My first play] *******************************************************************

TASK [Gathering Facts] *****************************************************************
ok: [192.0.2.50]
ok: [192.0.2.51]
ok: [192.0.2.52]

TASK [Ping my hosts] *******************************************************************
ok: [192.0.2.50]
ok: [192.0.2.51]
ok: [192.0.2.52]

TASK [Print message] *******************************************************************
ok: [192.0.2.50] => {
    "msg": "Hello world"
}
ok: [192.0.2.51] => {
    "msg": "Hello world"
}
ok: [192.0.2.52] => {
    "msg": "Hello world"
}

PLAY RECAP *****************************************************************************
192.0.2.50: ok=3  changed=0  unreachable=0  failed=0  skipped=0  rescued=0  ignored=0
192.0.2.51: ok=3  changed=0  unreachable=0  failed=0  skipped=0  rescued=0  ignored=0
192.0.2.52: ok=3  changed=0  unreachable=0  failed=0  skipped=0  rescued=0  ignored=0
```
В этом выводе можно увидеть:
- Названия, которые вы задаёте play и каждой задаче. Всегда используйте описательные имена — это упрощает проверку и отладку playbook’ов.
    
- Задача “Gathering Facts” выполняется неявно. По умолчанию Ansible собирает информацию об инвентаре, которую можно использовать в playbook.
    
- Статус каждой задачи. Каждая задача имеет статус `ok`, что означает успешное выполнение.
    
- Итоговый отчёт (play recap), который суммирует результаты всех задач для каждого хоста. В данном примере выполнено три задачи, поэтому `ok=3` указывает, что все три задачи успешно завершились.
