`networks` — это механизм, с помощью которого Docker Compose управляет виртуальными сетями между контейнерами.

`networks` позволяют:
- Соединять контейнеры в изолированные сетевые пространства.
- Управлять видимостью контейнеров друг для друга.
- Контролировать доступ к внешней сети.
- Задавать параметры IP-адресации, драйвера сети и другие настройки.

---
Без явного указания сетей:
- Все контейнеры по умолчанию попадают в одну общую сеть с именем `<имя_проекта>_default`.
- Они могут свободно общаться друг с другом по имени сервиса.

С явным указанием сетей:
- Вы можете создавать изолированные сети.    
- Чётко определять, кто к кому имеет доступ.
- Использовать разные драйверы (например, bridge, overlay).
- Разграничивать зоны безопасности.

---
**Простое указание сети для сервисов**
```yml
version: "3.8"

services:
  db:
    image: postgres
    networks:
      - backend

  app:
    image: myapp
    networks:
      - backend
      - frontend

  nginx:
    image: nginx
    networks:
      - frontend

networks:
  frontend:
  backend:
```
