`driver` определяет сетевой драйвер, то есть способ, которым Docker создаёт и управляет виртуальной сетью для контейнеров.

Фактически, драйвер указывает тип сети, её уровень изоляции, взаимодействия между контейнерами и поведение по отношению к хост-сети и внешним сетям.

| Драйвер   | Где используется                                 | Что делает                                                                     |
| --------- | ------------------------------------------------ | ------------------------------------------------------------------------------ |
| `bridge`  | По умолчанию для локальных сетей                 | Создаёт изолированную сеть на одном хосте. Контейнеры внутри видят друг друга. |
| `host`    | Локальные сети                                   | Контейнер использует сетевой стек хоста. Нет сетевой изоляции.                 |
| `overlay` | Кластер Docker Swarm                             | Создаёт сеть между контейнерами на разных хостах. Требует активного Swarm.     |
| `macvlan` | Когда требуется "видимость" на уровне сети хоста | Контейнер получает отдельный MAC и IP-адрес в сети хоста.                      |
| `none`    | Локальные сети                                   | Контейнер полностью изолирован от сети.                                        |
