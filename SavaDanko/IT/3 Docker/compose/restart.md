Параметр `restart` управляет политикой автоматического перезапуска контейнера при его остановке или сбое.

Эта настройка позволяет повысить отказоустойчивость и снизить количество ручных вмешательств.

| Значение            | Что означает                                                               |
| ------------------- | -------------------------------------------------------------------------- |
| `no` (по умолчанию) | Контейнер не перезапускается автоматически.                                |
| `always`            | Контейнер всегда перезапускается, независимо от причины остановки.         |
| `on-failure`        | Контейнер перезапускается только при ошибке (возврат код ≠ 0).             |
| `unless-stopped`    | Контейнер перезапускается всегда, кроме случая, когда его явно остановили. |

---

###### Примеры в `docker-compose.yml`
Перезапуск только при ошибке
```yaml
services:
  app:
    image: myapp
    restart: on-failure
```

Постоянный перезапуск, даже после ручного ребута сервера
```yaml
services:
  worker:
    image: myworker
    restart: always
```
