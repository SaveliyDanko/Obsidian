Параметр `build` позволяет прямо из `docker-compose.yml` собирать Docker-образ, а не использовать готовый образ из реестра.

Это особенно удобно:
- При разработке собственного приложения.
- Когда образ зависит от локального кода.
- Для автоматизации CI/CD процессов.
    
###### **Пример использования** `build`
```yaml
services:
  web:
    build: ./web
    ports:
      - "5000:5000"
```
Пояснение:
- Docker будет искать в папке `./web` файл `Dockerfile`.
- На основе этого файла соберётся новый образ.
- Из него запустится контейнер.
