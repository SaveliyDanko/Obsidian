`volumes` — это механизм для создания и подключения томов (постоянных хранилищ данных) к контейнерам, определённым в `docker-compose.yml`.

Локальное подключение тома к контейнеру (определение на уровне сервиса):
```yml
services:
  db:
    image: postgres
    volumes:
      - db_data:/var/lib/postgresql/data
```
Двоеточие `:` разделяет два пространства:
```yml
<что_подключаем_снаружи> : <куда_подключаем_внутрь_контейнера>
```
`db_data` — это имя тома, который создаётся или уже существует на стороне хоста (или управляется самим Docker).
`/var/lib/postgresql/data` — это путь внутри контейнера, куда будет смонтирован этот том.

Явное объявление томов в нижней части файла:
```yml
volumes:
  db_data:
```

Это эквивалент команды:
```bash
docker volume create db_data
```
