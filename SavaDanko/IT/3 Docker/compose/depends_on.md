Параметр `depends_on` описывает зависимости между сервисами, указывая, что один контейнер должен запускаться после другого.

Он регулирует порядок запуска, но НЕ гарантирует, что зависимый контейнер будет полностью готов к работе (например, что БД уже приняла соединения).

###### **Пример использования**
```yaml
services:
  app:
    image: myapp
    depends_on:
      - db

  db:
    image: postgres
```

В новых версиях Compose (формат 3.4+) можно использовать расширенный синтаксис с условиями:
```yaml
services:
  app:
    image: myapp
    depends_on:
      db:
        condition: service_healthy

  db:
    image: postgres
    healthcheck:
      test: ["CMD", "pg_isready", "-U", "postgres"]
      interval: 5s
      retries: 5
```
Пояснение:
- Контейнер `app` стартует только после того, как контейнер `db` стал "здоровым" согласно `healthcheck`.  
- Это гораздо надёжнее, чем просто ждать запуска процесса.
