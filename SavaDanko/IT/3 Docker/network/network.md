**Docker Network** — это виртуальная сеть, создаваемая и управляемая Docker для связи контейнеров между собой и с внешними сетями.

**Docker Bridge** — это тип Docker-сети по умолчанию, реализующий виртуальный сетевой мост (Эта команда разрешит имя в IP-адрес `registry.hub.docker.com`, потому что указанный домен поиска DNS автоматически дополнит имя хоста.bridge), через который контейнеры одного хоста могут обмениваться данными друг с другом и с внешним миром через интерфейс **docker0**.

**Closed containers** - это контейнер, который полностью запрещает любой сетевой трафик. Процессы, запущенные внутри такого контейнера, имеют доступ только к собственному интерфейсу loopback.
```zsh
docker run --rm --net none alpine:latest ip addr
```

**Bridge-контейнер** — это контейнер Docker, подключённый к виртуальной сети через мостовой интерфейс `docker0`, что позволяет ему:  
- Общаться с другими контейнерами на том же хосте  
- Выходить в сеть хоста и интернет (если настроен)
- Сохранять изоляцию за счёт отдельного приватного IP и namespace
Это наиболее распространённый и гибкий сетевой режим контейнеров Docker.

**Joined Containers** — это контейнеры Docker, которые разделяют один и тот же сетевой стек. Новый контейнер полностью использует сетевые интерфейсы уже работающего контейнера, включая все его IP-адреса, порты и маршруты.
```zsh
docker run -it --net container:<имя_или_ID_контейнера> alpine sh
```

**Open Containers** — это контейнеры Docker, которые полностью разделяют сетевой стек с хост-системой, работая напрямую в сети хоста без какой-либо сетевой изоляции.
