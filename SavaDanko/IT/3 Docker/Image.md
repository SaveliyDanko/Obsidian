**Docker image** можно представить как шаблон для запуска приложения — он содержит всё, что нужно для запуска: код, зависимости, библиотеки и настройки окружения.

Docker Image — это неизменяемый (immutable) слой, включающий:
- Файловую систему (например, Linux)
- Приложение или сервис
- Зависимости (например, библиотеки)
- Команды для запуска (например, `CMD`)
- Переменные окружения
- Метаданные (автор, теги, порты и т.д.)

Docker Image состоит из слоёв:
- Каждый слой — это изменение к предыдущему (например, установка пакета).
- Слои кэшируются, поэтому Docker может быстро пересобирать образы.
- Слои повторно используются другими образами, если они совпадают (экономия места).
