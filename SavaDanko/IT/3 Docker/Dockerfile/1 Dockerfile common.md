**Dockerfile** — это текстовый файл со специальными инструкциями, который описывает, как собрать Docker-образ.  
Он содержит пошаговый рецепт: какую операционную систему использовать, какие зависимости установить, какие файлы скопировать, какие команды выполнить и как запускать приложение внутри контейнера.

#### Пример простого Dockerfile
```dockerfile
# 1. Указываем базовый образ
FROM openjdk:17-jdk-slim

# 2. Устанавливаем рабочую директорию
WORKDIR /app

# 3. Копируем jar-файл в контейнер
COPY target/myapp.jar app.jar

# 4. Указываем команду запуска
ENTRYPOINT ["java", "-jar", "app.jar"]
```

#### Как работает Dockerfile?
Когда вы выполняете:
```bash
docker build -t myapp:1.0 .
```
Docker построчно читает Dockerfile и создает образ на основе этих инструкций. Каждая инструкция создает новый слой, который кэшируется — это ускоряет сборку и позволяет переиспользовать промежуточные результаты.

#### Основные инструкции в Dockerfile

| Инструкция           | Описание                                                                |
| -------------------- | ----------------------------------------------------------------------- |
| `FROM`               | Указывает базовый образ (например, `ubuntu`, `openjdk`, `node`)         |
| `WORKDIR`            | Устанавливает рабочую директорию внутри контейнера                      |
| `COPY`               | Копирует файлы из текущей директории на хосте внутрь контейнера         |
| `ADD`                | Похож на `COPY`, но может распаковывать архивы и загружать по URL       |
| `RUN`                | Выполняет команду во время сборки (например, `apt install`)             |
| `CMD` / `ENTRYPOINT` | Указывает, какую команду запускать при старте контейнера                |
| `ENV`                | Устанавливает переменные окружения                                      |
| `EXPOSE`             | Документирует порт, который будет открыт (не делает порт доступным сам) |
| `ARG`                | Устанавливает build-time переменные (в отличие от `ENV`)                |
| `LABEL`              | Добавляет метаданные (например, автор, версия и т.д.)                   |
