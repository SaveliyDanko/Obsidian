Union File System позволяет создавать write layer, в который можно сохранять данные, но при удалении container будет удален и write layer, что не позволит сохранить информацию.
Также у других container нет доступа к write layer в container.

Volumes в Docker позволяют сохранять данные программы на host file system, что позволяет сохранять данные даже после удаления container и удобно обмениваться данными между containers.
Можно создать пустой volume, в который container будет писать данные. Можно привязать volume к уже существующим данным на host file system.

**Volumes types:**
- bind mount - используют любой каталог или файл на хостовой операционной системе, который укажет пользователь.
- managed volume - используют каталоги, которые создаёт сам демон Docker в специальном пространстве, управляемом Docker, называемом Docker managed space

**Cleaning up Volumes**
Используйте флаг `-v` при удалении контейнеров, чтобы предотвратить накопление осиротевших томов.
```bash
docker rm -v student
```
