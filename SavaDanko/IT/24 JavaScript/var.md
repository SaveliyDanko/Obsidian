The `var` statement declares function-scoped or globally-scoped variables, optionally initializing each to a value.

Область видимости переменной, объявленной через `var`, ограничивается одной из следующих фигурно-скобочных конструкций, в зависимости от того, какая наиболее близко окружает `var`:
- Тело функции
- Блок статической инициализации
    
Если вышеуказанные случаи не применимы:
- Используется область текущего модуля, если код выполняется в module mode
- Или глобальная область, если код выполняется в script mode

```js
function foo() {
  var x = 1; // переменная x определена в области функции foo
  function bar() {
    var y = 2; // переменная y определена в области функции bar
    console.log(x); // => 1 (bar имеет доступ к x из внешней функции foo)
    console.log(y); // => 2 (y определена в bar и используется внутри неё)
  }
  bar(); // вызов функции bar
  console.log(x); // => 1 (x всё ещё в области foo)
  console.log(y); // ReferenceError (y определена только в bar, вне её недоступна)
}
foo(); // вызов функции foo
```
- `var` не имеет блочной области видимости, а имеет функциональную.
- Переменная `x`, объявленная в `foo`, доступна внутри `bar`.
- Переменная `y`, объявленная внутри `bar`, не доступна вне `bar`.

**Другие блочные конструкции** — такие как:
- `block statements` (например, `{ ... }` вне функции)
- `try...catch`
- `switch`
- заголовки в `for`, `for...in`, `for...of`
не создают область видимости для `var`.

Переменные, объявленные с `var` внутри таких блоков, видимы за пределами этих блоков. Это может привести к ошибкам, если ожидать блочной области видимости, как с `let` или `const`.
