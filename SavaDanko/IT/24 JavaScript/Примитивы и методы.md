#### 1. Примитивные типы данных
Примитивные типы данных включают:
- `string` (строка)
- `number` (число)
- `bigint` (большое число)
- `boolean` (логический тип)
- `symbol` (уникальный идентификатор)
- `null` (пустое значение)
- `undefined` (значение не определено)

Эти типы являются примитивами, потому что они не являются объектами и работают как простые значения. Например, строка `'hello'` не обладает методами сама по себе.

---
#### 2. Методы примитивов
Примитивы, несмотря на отсутствие методов, могут использоваться как объекты. Это достигается благодаря временным объектным-оберткам, которые создаются автоматически.

Пример:
```javascript
let str = "Hello";
console.log(str.toUpperCase()); // HELLO
```
- В момент вызова `str.toUpperCase()` создаётся объектная обёртка `String`.
- Объект получает метод `toUpperCase` и выполняет его.
- После выполнения обёртка удаляется.

Для каждого примитива существует соответствующий объект:
- `String` для строк
- `Number` для чисел
- `Boolean` для логических значений
- `BigInt` для больших чисел
- `Symbol` для символов

---

#### 3. Особенности объектных обёрток
- Они создаются временно и удаляются сразу после выполнения.
- Методы примитивов работают быстро и не потребляют лишнюю память, потому что объектная обёртка существует только на время вызова метода.

Пример работы:
```javascript
let str = "Hello";

console.log(str.toUpperCase()); // HELLO
console.log(typeof str); // string

let num = 123.45;
console.log(num.toFixed(1)); // 123.5
```
Объектная обёртка используется только в момент вызова методов. Примитивы остаются неизменными.

---
#### 4. Ключевые отличия примитивов от объектов
1. Примитивы неизменяемы. 
    ```javascript
    let str = "Hello";
    str[0] = "h"; // Ошибка, ничего не изменится
    console.log(str); // Hello
    ```    
    В отличие от объектов, нельзя напрямую изменить отдельные части примитива.
    
2. Примитивы занимают меньше памяти. Объекты, даже если их функциональность схожа с примитивами, более ресурсоёмки.
    

---
#### 5. `null` и `undefined`
- У `null` и `undefined` нет методов.
- Попытка вызова метода вызывает ошибку:
    ```javascript
    let value = null;
    console.log(value.toString()); // Ошибка: Cannot read properties of null
    ```
    

---
#### 6. Явное использование объектных обёрток
Объекты, соответствующие примитивам (`String`, `Number`, `Boolean`), можно создавать явно, но это плохая практика.

Пример:
```javascript
let strObj = new String("Hello");
console.log(typeof strObj); // object

let numObj = new Number(123);
console.log(typeof numObj); // object
```

Создание таких объектов приводит к путанице:
```javascript
console.log(new String("Hello") === "Hello"); // false
```

Использовать такие объекты вместо примитивов не рекомендуется.

---
#### 7. Исключения при работе с примитивами
Примитивы могут быть преобразованы в объекты, когда используются в контексте, где ожидается объект.

Пример:
```javascript
let zero = new Number(0);

if (zero) { 
    console.log("zero is truthy!"); // Выполняется
}
```
Это связано с тем, что объект всегда `truthy` независимо от значения, а примитив может быть `falsy` (например, `0`, `''`, `null`).
