**`Event`** — это базовый интерфейс в DOM API, который описывает любое событие: пользовательское (клик, нажатие клавиши, ввод) или программное (созданное скриптом).  
Все специализированные события (`MouseEvent`, `KeyboardEvent`, `FocusEvent`, `CustomEvent` и др.) наследуются от `Event`.

###### **Иерархия наследования**
```
Event
│
├── UIEvent
│    ├── MouseEvent
│    ├── KeyboardEvent
│    ├── FocusEvent
│    └── WheelEvent
│
├── ClipboardEvent
├── DragEvent
├── AnimationEvent
├── TransitionEvent
├── CustomEvent
...
```
`Event` — это “родитель”, который хранит общую информацию и поведение для всех типов событий.

###### **Как создаётся объект `Event`**
Браузером (автоматически)
Когда пользователь кликает, двигает мышь, скроллит, печатает, браузер создаёт объект события и передаёт его обработчику как первый аргумент:
```js
document.addEventListener('click', function(e) {
  console.log(e instanceof Event); // true
});
```

Программно (создание вручную)
```js
const event = new Event('build', { bubbles: true, cancelable: true });
element.dispatchEvent(event);
```

###### **Основные свойства `Event`**
- `bubbles` — возвращает, является ли событие всплывающим.
- `cancelBubble` — задаёт или возвращает, должно ли событие всплывать вверх по иерархии.
- `cancelable` — возвращает, можно ли предотвратить стандартное действие события.
- `composed` — возвращает, является ли событие составным (composed) или нет.
- `currentTarget` — возвращает элемент, на котором сработал текущий обработчик события.
- `defaultPrevented` — возвращает, был ли вызван метод `preventDefault()` для данного события.
- `eventPhase` — возвращает фазу обработки события (capturing, target или bubbling).
- `isTrusted` — возвращает, было ли событие вызвано пользователем (`true`) или создано скриптом (`false`).
- `target` — возвращает элемент, на котором произошло событие.
- `timeStamp` — возвращает время (в миллисекундах от эпохи Unix), когда событие было создано.
- `type` — возвращает имя события (например, `"click"`, `"input"` и т.д.).

###### **Методы `Event`**
- `createEvent()` — создаёт новое событие.
- `composedPath()` — возвращает путь события (массив элементов, через которые оно прошло).
- `preventDefault()` — отменяет событие, если оно может быть отменено, предотвращая выполнение стандартного действия браузера.
- `stopImmediatePropagation()` — предотвращает выполнение других обработчиков этого же события на текущем элементе.
- `stopPropagation()` — предотвращает дальнейшее всплытие события по дереву DOM.
