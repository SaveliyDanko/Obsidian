NASM (Netwide Assembler) — это мощный ассемблер для архитектур x86 и x86-64, который поддерживает множество флагов и атрибутов для управления процессом компиляции. Эти флаги позволяют указывать формат выходного файла, архитектуру, задавать макросы, управлять оптимизацией и выводом сообщений. Рассмотрим основные флаги и атрибуты, которые можно использовать с NASM.

### ==1. **Флаг `-f` (формат выходного файла)**==
Флаг `-f` определяет, в каком формате будет сгенерирован объектный файл. Некоторые поддерживаемые форматы:
- **`elf32`** — 32-битный формат ELF (Linux).
- **`elf64`** — 64-битный формат ELF (Linux).
- **`bin`** — бинарный формат (raw machine code, без заголовков).
- **`macho32`** — 32-битный формат Mach-O (macOS).
- **`macho64`** — 64-битный формат Mach-O (macOS).
- **`win32`** — 32-битный объектный файл для Windows.
- **`win64`** — 64-битный объектный файл для Windows.
- **`obj`** — 16-битный объектный файл (формат DOS).
- **`wasm`** — объектный файл в формате WebAssembly.

Пример:
```bash
nasm -f elf64 hello.asm -o hello.o
```

### ==2. **Флаг `-o` (имя выходного файла)**==
Флаг `-o` задаёт имя для выходного файла, который будет создан после компиляции. Без этого флага по умолчанию используется имя исходного файла с расширением `.o`.

Пример:
```bash
nasm -f bin -o output.bin program.asm
```

### 3. ==**Флаг `-l` (файл листинга)**==
Флаг `-l` генерирует файл листинга, который включает исходный код программы вместе с сгенерированным машинным кодом, смещениями и адресами. Это полезно для отладки и анализа программы.

Пример:
```bash
nasm -f elf32 hello.asm -l hello.lst
```

### ==4. **Флаг `-D` (макросы)**==
С помощью флага `-D` можно определять макросы, которые затем могут быть использованы в коде. Это похоже на использование директивы `#define` в языке C.

Пример:
```bash
nasm -f elf64 -DDEBUG=1 hello.asm -o hello.o
```

В программе можно использовать:
```nasm
%ifdef DEBUG
    ; код для отладки
%endif
```

### ==5. **Флаг `-I` (каталог для включаемых файлов)**==
Флаг `-I` указывает каталоги, в которых NASM будет искать включаемые файлы с директивой `INCLUDE`.

Пример:
```bash
nasm -I /path/to/include/ -f elf32 program.asm -o program.o
```

### ==6. **Флаг `-w` (управление предупреждениями)**==
Флаг `-w` управляет тем, какие предупреждения компилятор NASM будет отображать или игнорировать. Он позволяет включить или отключить определённые категории предупреждений.

- **`-w+error`** — отображать все предупреждения как ошибки.
- **`-w+all`** — включить все предупреждения.
- **`-w-label-orphan`** — управляет предупреждениями о метках, которые могут быть синтаксически некорректными.
- **`-w+orphan-labels`** — разрешить метки без двоеточий.

Пример отключения предупреждений:
```bash
nasm -w+all -f elf32 hello.asm -o hello.o
```

### 7. ==**Флаг `-g` (информация для отладки)**==
Флаг `-g` включает генерацию информации для отладки, что полезно при использовании отладчиков, таких как GDB. Этот флаг генерирует символы и информацию о исходных строках для объектного файла.

Пример:
```bash
nasm -g -f elf64 hello.asm -o hello.o
```

### 8. ==**Флаг `-E` (предварительная обработка)**==
Флаг `-E` заставляет NASM выполнить только предварительную обработку (аналогично `cpp` в C). Это позволяет увидеть результат обработки макросов и включаемых файлов перед компиляцией.

Пример:
```bash
nasm -E program.asm
```

### ==9. **Флаг `-X` (выбор синтаксиса)**==
Флаг `-X` позволяет выбрать синтаксис для символов локальных меток. По умолчанию NASM использует символ `.` для локальных меток, но вы можете изменить его.

Пример:
```bash
nasm -X gnu -f elf64 hello.asm
```

### ==10. **Флаг `-F` (формат отладочной информации)**==
Флаг `-F` позволяет указать формат отладочной информации (если используется флаг `-g`). Примеры форматов: `dwarf`, `stabs`, `dwarf2`.

Пример:
```bash
nasm -g -F dwarf -f elf64 hello.asm -o hello.o
```

### ==11. **Флаг `-P` (предварительная вставка файла)**==
Флаг `-P` позволяет вставить указанный файл в начале компиляции перед обработкой исходного файла. Это похоже на директиву `INCLUDE`, но она выполняется перед обработкой основного файла.

Пример:
```bash
nasm -P macros.inc -f elf64 program.asm
```

### ==12. **Флаг `-v` (версия компилятора)**==
Флаг `-v` выводит информацию о версии NASM и завершает выполнение. Полезно для проверки установленной версии.

Пример:
```bash
nasm -v
```

### ==Заключение==
NASM поддерживает множество флагов, которые могут быть полезны при написании программ на ассемблере, от простого управления выходным форматом до сложных макросов и отладочной информации. Использование правильных флагов помогает сделать процесс разработки более гибким и удобным, особенно при отладке или работе с разными платформами.
