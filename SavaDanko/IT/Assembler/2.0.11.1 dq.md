`DQ` (Define Quadword) — это директива в языке ассемблера, используемая для резервирования памяти и инициализации данных с размером 8 байт (64 бита). Эта директива обычно применяется для хранения 64-битных значений, таких как большие целые числа, адреса или указатели.

### ==Синтаксис==
```
DQ <значение1>, <значение2>, ...
```
Где:
- `<значение1>`, `<значение2>`, ... — это значения, которые будут сохранены в виде 64-битных данных в памяти.

### ==Пример использования==
```asm
section .data
    my_number dq 1234567890123456789  ; Определение числа 64-бит (quadword)
    array dq 1, 2, 3, 4               ; Массив из 64-битных чисел
```
В данном примере:
- `my_number` резервирует 8 байт в памяти и инициализируется значением `1234567890123456789`.
- `array` резервирует 32 байта (4 значения по 8 байт каждое) и инициализируется четырьмя числами.

### ==Использование в программах==
64-битные значения, определённые директивой `DQ`, могут использоваться в арифметических операциях, адресных вычислениях и указателях. Особенно полезно в 64-битных системах, таких как x86-64 архитектура, где регистры и адреса могут занимать 64 бита.

### ==Чтение и запись==
Для доступа к 64-битным данным, можно использовать 64-битные регистры процессора, такие как `RAX`, `RBX`, `RCX` и т.д. Пример:
```asm
section .data
    num dq 0x1122334455667788         ; Определяем 64-битное число

section .text
    global _start

_start:
    mov rax, [num]                    ; Читаем 64-битное значение в регистр RAX
    ; Дальнейшие операции
```
В этом примере значение из памяти, определённое с помощью `dq`, загружается в регистр `RAX`.
