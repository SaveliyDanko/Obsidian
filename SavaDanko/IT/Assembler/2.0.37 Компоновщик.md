./**Компоновщик** — это важная часть процесса сборки программы на ассемблере (как и на других языках программирования). Его задача — взять несколько **объектных файлов** (полученных после компиляции) и собрать их в один **исполняемый файл**, который можно запустить на компьютере. Компоновщик объединяет код, данные, таблицы символов и другие секции, а также решает все ссылки между ними.

### ==Основные функции компоновщика:==
1. **Связывание объектных файлов**:
   Если программа состоит из нескольких модулей, каждый из которых был скомпилирован в отдельный объектный файл, компоновщик объединяет их в один файл. Он также разрешает ссылки на функции и данные, которые определены в разных файлах.

2. **Создание исполняемого файла**:
   Компоновщик объединяет объектные файлы, необходимые для программы, и создаёт один исполняемый файл. Это может быть файл формата ELF на Linux или PE на Windows.

3. **Обработка таблицы символов**:
   В процессе компиляции создаётся **таблица символов**, которая хранит информацию о переменных, функциях и других именованных объектах программы. Компоновщик использует таблицу символов для разрешения ссылок на эти объекты.

4. **Обработка таблицы релокаций**:
   Если программа ссылается на переменные или функции, чьи адреса ещё не известны на этапе компиляции (например, функция, определённая в другом файле), компоновщик использует **таблицу релокаций** для обновления этих ссылок, когда программа загружается в память.

### ==Пример использования компоновщика:==
На этапе компиляции каждая часть программы превращается в объектный файл. Компоновщик объединяет их в исполняемый файл:
```bash
ld -o hello hello.o
```
Здесь компоновщик создаёт исполняемый файл `hello` из объектного файла `hello.o`.

### ==Таблица символов==
Таблица символов — это структура данных, которая хранит информацию о метках и символах программы после компиляции. Она помогает компоновщику понять, где находятся определённые функции или данные, и как связать их между объектными файлами.

Пример программы:
```asm
global _start
section .data
message: db 'Hello, world!', 10

section .text
_start:
    mov rsi, message
```
Таблица символов для этой программы может включать такие элементы, как:
- Адрес метки `_start`.
- Адрес переменной `message`.

Команда `nm` может быть использована для просмотра содержимого таблицы символов:
```bash
nm symbols.o
```

Вывод может выглядеть так:
```
00000000 T _start
00000000 d message
```
