Компилятор в ассемблере — это инструмент, который преобразует исходный код программы, написанной на ассемблере, в набор машинных инструкций, понятных процессору. Процесс компиляции в ассемблере достаточно "прозрачный", потому что каждая команда ассемблера обычно непосредственно соответствует одной или нескольким машинным инструкциям.

### ==Основные функции компилятора в ассемблере:==
1. **Преобразование в машинный код**: Компилятор ассемблера переводит текстовые инструкции (например, `mov`, `add`, `sub`) в машинные коды (биты и байты), которые может напрямую исполнять процессор.
2. **Работа с модулями**: Компилятор также поддерживает работу с программами, которые разделены на несколько файлов-модулей. В этом случае файлы компилируются независимо друг от друга и затем связываются вместе на этапе компоновки.

### ==Работа с несколькими модулями==
Когда программа разбита на несколько файлов, которые должны взаимодействовать друг с другом, необходимо указывать, какие функции или данные из одного файла доступны в другом.

1. **`global`** — используется для того, чтобы объявить функцию или переменную, доступную другим модулям программы. Это аналог экспорта.
2. **`extern`** — указывает, что функция или переменная находится в другом модуле и её нужно импортировать для использования.

### ==Пример использования:==
- **Файл `a.asm`** содержит две функции `f` и `h`. Функция `f` экспортируется с помощью директивы `global`, а функция `h` остаётся локальной для файла и недоступной другим модулям.
  
- **Файл `b.asm`** импортирует функцию `f` с помощью директивы `extern` и может вызывать её, как показано в коде.

### ==Пример:==
```asm
; a.asm
global f

f:
    ; код функции f

h:
    ; код функции h (недоступна для других файлов)
```

```asm
; b.asm
extern f

call f  ; вызов функции f
```

### ==Заголовочные файлы и интерфейсы==
При работе с библиотеками, состоящими из нескольких функций, лучше всего использовать заголовочные файлы (например, `lib.inc`), чтобы упростить импорт и экспорт функций между модулями. Заголовочные файлы включают определения `extern` для всех функций, которые могут быть использованы в других модулях.

Пример заголовочного файла:
```asm
; lib.inc
extern f1
extern f2
extern f3
```
Когда библиотека расширяется, вместо изменения каждого файла, где используются её функции, достаточно обновить только заголовочный файл.
