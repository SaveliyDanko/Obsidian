В ассемблере x86-64 и других ассемблерах существуют две основные формы меток: с точкой в начале и без точки.

### ==Метки без точки==
**Метки без точки** являются обычными метками, которые используются для обозначения местоположений в коде. Эти метки могут быть использованы для переходов, вызовов и других операций.

Пример:
```asm
section .text
    global _start

_start:
    mov rdi, rsi
    jmp my_label

my_label:
    mov rax, 60
    mov rdi, 0
    syscall
```
В этом примере `my_label` — это обычная метка. Она используется для обозначения места в коде, к которому можно перейти с помощью инструкций, таких как `jmp`, `call` и других.

### ==Метки с точкой==
**Метки с точкой** используются в некоторых ассемблерах (например, в NASM) для обозначения локальных меток. Эти метки видны только в пределах секции или блока, в котором они определены, и предназначены для предотвращения конфликтов имен с другими метками. Они обычно используются внутри макросов или секций кода для создания уникальных имен, которые не будут конфликтовать с другими метками в проекте.

Пример:
```asm
section .text
    global _start

_start:
    jmp .local_label

    ; Пример макроса с локальными метками
    %macro example 0
        mov rax, 1
        jmp .local_macro_end
    %endmacro

    ; Вызов макроса
    example

.local_label:
    mov rax, 60
    mov rdi, 0
    syscall

.local_macro_end:
    ; Метка, до которой будет выполняться переход после вызова макроса
```

### ==Комбинированные метки==
Можно обращаться к локальным меткам из другой области видимости, если прописывать их полные имена: 
`_start.loop`

### ==Основные различия==
1. **Область видимости**:
   - **Без точки**: Метки без точки имеют глобальную видимость в пределах модуля или секции, где они определены. Они могут быть использованы в любом месте после их определения.
   - **С точкой**: Метки с точкой обычно имеют локальную видимость, что означает, что они доступны только в пределах секции или блока кода, где они определены. Это позволяет избежать конфликтов имен с другими метками в коде.

2. **Использование**:
   - **Без точки**: Обычно используются для общих целей, таких как переходы, вызовы функций и другие операции.
   - **С точкой**: Часто используются внутри макросов и других структур кода для создания уникальных, локальных меток, которые не пересекаются с метками в других частях программы.
