Команда `mov` в NASM (Netwide Assembler) является одной из самых базовых и часто используемых инструкций в языке ассемблера. Она предназначена для передачи данных из одного места в другое. Эта команда используется для копирования значений между регистрами, памятью и непосредственными (немедленными) значениями.

### ==Синтаксис команды `mov`==
Основной синтаксис команды `mov` выглядит следующим образом:
```asm
mov destination, source
```
где:
- `destination` — место, куда нужно скопировать данные (регистры или ячейки памяти).
- `source` — источник данных, который может быть регистром, ячейкой памяти или немедленным значением.

### ==Примеры использования команды `mov`==
1. **Перемещение значения между регистрами**
```asm
mov rax, rbx    ; Копирует значение из регистра rbx в регистр rax
```

2. **Перемещение значения из немедленного значения в регистр**
```asm
mov rax, 5      ; Помещает значение 5 в регистр rax
```

3. **Перемещение значения из регистра в ячейку памяти**
```asm
mov [memory], rax  ; Копирует значение из регистра rax в память по адресу memory
```

4. **Перемещение значения из ячейки памяти в регистр**
```asm
mov rax, [memory]  ; Копирует значение из памяти по адресу memory в регистр rax
```

5. **Перемещение значения из немедленного значения в ячейку памяти**
```asm
mov [memory], 10   ; Помещает значение 10 в память по адресу memory
```

### ==Особенности команды `mov`==
1. **Размер операндов**
   Операнды в команде `mov` должны быть одного размера. Например, нельзя переместить 32-битное значение в 64-битный регистр или наоборот.
   ```asm
   mov eax, ebx   ; Допустимо: оба 32-битных регистра
   mov rax, rbx   ; Допустимо: оба 64-битных регистра
   ```

2. **Не поддерживается перемещение данных между двумя ячейками памяти**
   Нельзя использовать команду `mov` для перемещения данных непосредственно между двумя ячейками памяти. Нужно использовать регистр в качестве промежуточного шага:
```asm
mov rax, [source]   ; Загрузить данные из источника в регистр rax
mov [destination], rax ; Сохранить данные из регистра rax в назначение
```

3. **Немедленные значения**
   Немедленные значения (константы) могут быть загружены только в регистры или память, но не могут быть перемещены напрямую в другие немедленные значения:
   ```asm
   mov rax, 0xFF   ; Верно
   mov rbx, 0xFF   ; Верно, если rbx и rax оба 64-битные
   ```

4. **Заполнение и обрезка**
   При перемещении данных между регистрами разного размера, более крупный регистр заполняется нулями (для операций, связанных с типами данных с плавающей точкой и фиксированными числами).
```asm
mov al, 0xFF   ; Загружает 0xFF в младшие 8 бит регистра rax, остальные биты очищаются
```
