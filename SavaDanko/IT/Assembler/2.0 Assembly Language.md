## ==Setting Up the Environment==
Если вы хотите приступить к программированию на ассемблере с использованием указанной среды, вот краткий список действий, которые помогут вам настроить систему:

1. **Операционная система:**
   - Рекомендуется использовать **Debian GNU/Linux 8.0**, но вы можете выбрать любую дистрибуцию Linux. Если используете Windows, то вам подойдет **Windows Subsystem for Linux (`WSL`)**.

2. **Компиляторы и инструменты:**
   - **`NASM (Netwide Assembler)`** — ассемблерный компилятор. Установите его с помощью:
     ```bash
     sudo apt-get install nasm
     ```
   - **`GCC (GNU Compiler Collection)`** — компилятор для языка C. Версия 4.9.2:
     ```bash
     sudo apt-get install gcc
     ```
   - **`GNU Make`** — система сборки:
     ```bash
     sudo apt-get install make
     ```
   - **`GDB (GNU Debugger)`** — отладчик, который поможет шаг за шагом исследовать выполнение программы:
     ```bash
     sudo apt-get install gdb
     ```

3. **Текстовый редактор:**
   - Выберите любой текстовый редактор с подсветкой синтаксиса, например, **`ViM`**. Чтобы установить его:
     ```bash
     sudo apt-get install vim
     ```

4. **Запуск и настройка Windows Subsystem for Linux (`WSL`):**
   - Если вы используете Windows, установите `WSL` и необходимые пакеты с помощью `apt-get`:
     - Официальное руководство по установке `WSL` можно найти [здесь](https://msdn.microsoft.com/en-us/commandline/wsl/install_guide).
     - Установите необходимые пакеты через команду:
       ```bash
       sudo apt-get install nasm gcc make gdb vim
       ```

5. **Дополнительные ресурсы:**
   - На сайте издательства `Apress` (по ссылке: http://www.apress.com/us/book/9781484224021) можно найти виртуальные машины с предустановленными инструментами для программирования на ассемблере и C. Виртуальные машины доступны как с графической оболочкой, так и с минимальной установкой, к которой можно подключиться через SSH.
   - В репозитории GitHub содержатся примеры кода из книги, ответы на вопросы и решения задач.

6. **`GDB (GNU Debugger)`:**
   - Используйте **`gdb`** для пошаговой отладки ассемблерных программ. Это поможет лучше понять логику выполнения программ. Ознакомьтесь с приложением A в книге для быстрого введения в **`gdb`**.

7. **Компиляция и запуск примеров:**
   - Вы можете компилировать код из книги и запускать его с использованием **`NASM`** и **GCC**.
   - Например, для компиляции ассемблерного файла:
     ```bash
     nasm -f elf64 my_program.asm
     gcc -o my_program my_program.o
     ./my_program
     ```

Эти шаги помогут вам создать среду для работы с ассемблером и C. Не стесняйтесь обращаться к документации и использовать отладочные инструменты для более глубокого понимания кода.
