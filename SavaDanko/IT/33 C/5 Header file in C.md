Header-файл (`.h`) в C — это файл с объявлениями, которые должны быть видны из других `.c` файлов: типы, прототипы функций, константы/макросы, объявления глобальных переменных (`extern`). А `.c` файл обычно содержит реализацию.

###### **Что обычно лежит в** `.h`
1. Прототипы функций
```c
int add(int a, int b);
```

2. Типы и структуры
```c
typedef struct {
  int x, y;
} Point;
```

3. Константы/макросы/enum
```c
#define BUF_SIZE 1024
enum { OK = 0, ERR = 1 };
```


###### **Зачем разделять на** `.c` **и** `.h`
###### Разделение интерфейс vs реализация
`.h` — контракт: что доступно снаружи.
`.c` — как это устроено внутри.

Это позволяет менять реализацию в `.c` (оптимизация, другой алгоритм), не трогая код, который просто вызывает функции — пока интерфейс в `.h` не меняется.

###### Компиляция по частям
C компилирует каждый `.c` отдельно в объектный файл `.o`, потом линкер всё собирает.  
Если ты изменил только `file1.c`, пересобирается только он, а не весь проект.
Если же всё в одном файле, любая правка заставляет пересобирать всё.

###### Повторное использование модулей
Хочешь использовать модуль в другом проекте:
- берёшь `stack.c` + `stack.h`
- подключаешь `#include "stack.h"` и линкуешь `stack.c`
Удобно как библиотека.

###### Сокрытие деталей (инкапсуляция)
В `.h` можно оставить только то, что нужно пользователю модуля, а всё остальное спрятать в `.c`.
