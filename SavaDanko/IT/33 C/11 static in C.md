###### `static` **у локальной переменной внутри функции**
Хранится всю программу, но видна только в этом блоке/функции.
```c
#include <stdio.h>

void f(void) {
    static int count = 0; // инициализируется один раз
    count++;
    printf("%d\n", count);
}
```
- `count` не создаётся заново при каждом вызове, а сохраняет значение между вызовами.
- Обычно лежит в `.data/.bss`, а не на стеке.

###### `static` **у глобальной переменной (на уровне файла)**
Это означает internal linkage: переменная видна только внутри этого `.c` файла.
```c
// file1.c
static int g_state = 42; // нельзя использовать из других .c
```
Если в другом файле сделать `extern int g_state;` — не сработает, линкер не найдёт (она приватная для файла).

###### **Инициализация**
- `static` объекты инициализируются один раз.
- Если инициализатора нет — они обнуляются по умолчанию.
```c
static int x; // будет 0
```
