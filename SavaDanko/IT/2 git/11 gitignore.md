Файл `.gitignore` указывает Git, какие файлы и каталоги не нужно отслеживать. Это помогает не засорять репозиторий временными, системными или конфиденциальными файлами (например, `*.log`, `.env`).

### **Как работает:**
- Git **не добавит** в индекс файлы, указанные в `.gitignore`, если они **ещё не отслеживаются**.
- Если файл **уже отслеживается**, `.gitignore` его **не исключит** — нужно использовать `git rm --cached file`.
- Правила применяются **относительно расположения файла `.gitignore`**.
- Можно использовать **глобальный .gitignore** для игнорирования файлов во всех проектах (например, `.DS_Store`, `*.swp`).

#### **Синтаксис правил:**

| Шаблон           | Значение                                                               |
| ---------------- | ---------------------------------------------------------------------- |
| `*.log`          | Все файлы с расширением `.log`                                         |
| `build/`         | Папка `build` и всё внутри неё                                         |
| `!important.log` | **Не игнорировать** этот файл, даже если правило выше говорит обратное |
| `temp?.txt`      | Совпадение с `temp1.txt`, `temp2.txt` и т.д.                           |
| `/secret.txt`    | Только в корне проекта                                                 |
| `**/cache/`      | Все папки `cache` на любом уровне вложенности                          |
