#### ==**Отличие Git от других СКВ**==
Git сохраняет снимки всей файловой системы, а не разницу между версиями. При неизменности файла — создаётся ссылка, а не копия.

#### ==**Целостность данных в Git**==
Каждый объект хешируется (SHA-1) и идентифицируется по хешу. Это гарантирует, что подмена данных невозможна без обнаружения.

#### ==**Git не удаляет, а добавляет**==
Все действия — это добавление новых данных, поэтому восстановить старые состояния легко, а потерять — сложно.

#### ==**Три состояния файлов в Git**==
1. Modified — изменён, но не подготовлен к коммиту.
2. Staged — добавлен в индекс (`git add`).
3. Committed — зафиксирован в истории (`git commit`).

#### ==**Структура работы Git**==
1. Рабочая копия — текущие редактируемые файлы.
2. Область индексирования — подготовленные к коммиту изменения.
3. Каталог Git — база данных всех коммитов и объектов проекта.
