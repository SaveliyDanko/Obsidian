`tag` — это способ зафиксировать определённый момент в истории проекта, например, выпуск релиза. Это как "закладка" на конкретный коммит, чтобы к нему можно было легко вернуться.

##### Лёгкий тег (lightweight tag)
Просто метка на коммит, как ссылка.
```bash
git tag v1.0
```

##### Аннотированный тег (annotated tag)
Хранит: имя автора, дату, сообщение, может быть подписан GPG.
```bash
git tag -a v1.0 -m "First stable release"
```

По умолчанию `git push` не отправляет теги. Нужно явно указать:
```bash
git push origin v1.0
```
или все теги сразу:
```bash
git push --tags
```

##### Просмотр тегов
```bash
git tag
```

##### Удаление тега
Локально:
```bash
git tag -d v1.0
```
на удалённом репозитории:
```bash
git push origin --delete tag v1.0
```
