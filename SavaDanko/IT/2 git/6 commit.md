`git commit` сохраняет снимок текущего состояния проекта (из индексированной области) в историю версий. 

| Флаг                | Назначение                                                                                                                    |
| ------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| `-m "сообщение"`    | Указывает **сообщение коммита** напрямую в командной строке.                                                                  |
| `-a`                | Автоматически **индексирует (добавляет)** все отслеживаемые файлы перед коммитом (аналог `git add` для отслеживаемых файлов). |
| `--amend`           | **Изменяет последний коммит**: можно скорректировать сообщение или добавить новые изменения.                                  |
| `--no-edit`         | Используется с `--amend`, чтобы оставить **старое сообщение коммита** без изменения.                                          |
| `--allow-empty`     | Создаёт **пустой коммит** (без изменений), полезно для триггеров или тестов.                                                  |
| `--squash <commit>` | Подготавливает объединение нескольких коммитов в один (используется вместе с `rebase`).                                       |
| `--verbose`         | Показывает diff вместе с сообщением коммита в редакторе (при коммите без `-m`).                                               |
