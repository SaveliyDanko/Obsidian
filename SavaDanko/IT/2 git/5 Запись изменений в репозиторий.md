#### ==Файлы в Git делятся на две категории:==
###### 1. Отслеживаемые файлы (tracked)
- Уже были включены в репозиторий.
- Могут быть неизменёнными, изменёнными или подготовленными к коммиту.

###### 2️. Неотслеживаемые файлы (untracked)
- Не находятся под версионным контролем (новые файлы, не добавленные в индекс).
- Git не следит за их изменениями, пока они не будут добавлены в индекс (`git add`).

### ==Цикл работы с файлами в Git==
1. **Редактируете файлы** → они становятся **изменёнными (modified)**.  
2. **Добавляете изменения в индекс (`git add`)** → файлы переходят в **статус staged**.  
3. **Фиксируете коммит (`git commit -m "описание"`)** → создаётся снимок состояния.  
4. **Повторяете цикл** при новых изменениях.
