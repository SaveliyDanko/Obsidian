Команда `git config` используется для настройки поведения Git: имени пользователя, почты, редактора, цветов, алиасов и других параметров.

#### **Уровни конфигурации:**

| Уровень        | Флаг             | Где сохраняется                  | Применяется к              |
| -------------- | ---------------- | -------------------------------- | -------------------------- |
| **Системный**  | `--system`       | `/etc/gitconfig`                 | Всем пользователям системы |
| **Глобальный** | `--global`       | `~/.gitconfig`                   | Текущему пользователю      |
| **Локальный**  | _(по умолчанию)_ | `.git/config` внутри репозитория | Только текущему проекту    |

> Приоритет: локальный > глобальный > системный

#### **Часто используемые настройки:**

| Настройка            | Назначение                                              |
| -------------------- | ------------------------------------------------------- |
| `user.name`          | Имя пользователя в коммитах                             |
| `user.email`         | Почта автора коммитов                                   |
| `core.editor`        | Редактор по умолчанию (например, `nano`, `vim`, `code`) |
| `color.ui`           | Включение цветного вывода (`auto`, `true`, `false`)     |
| `alias.имя`          | Создание псевдонима для команды Git                     |
| `init.defaultBranch` | Устанавливает имя ветки по умолчанию (`main`, `master`) |


#### **Особенности:**
- Git хранит конфигурацию в текстовых файлах, которые можно редактировать вручную.
- Команда `git config` может использоваться как для чтения, так и для записи настроек.
- Можно использовать `--list` для просмотра всех текущих настроек.
