Команда `git log` отображает историю коммитов в текущей ветке. Это основной инструмент для анализа того, кто, когда и какие изменения внёс в проект.

| Флаг                                             | Назначение                                                               |
| ------------------------------------------------ | ------------------------------------------------------------------------ |
| `--oneline`                                      | Показывает каждый коммит в **одной строке** (короткий хеш + сообщение).  |
| `--graph`                                        | Добавляет **ASCII-граф** ветвлений и слияний. Отлично с `--oneline`.     |
| `--decorate`                                     | Показывает метки (`HEAD`, имена веток, теги) рядом с коммитами.          |
| `--all`                                          | Показывает историю **всех веток**, а не только текущей.                  |
| `--author="имя"`                                 | Фильтрует коммиты по автору.                                             |
| `--since="2 weeks ago"` / `--until="2024-12-31"` | Фильтр по времени.                                                       |
| `-p`                                             | Показывает `diff` для каждого коммита.                                   |
| `--stat`                                         | Отображает статистику: сколько строк добавлено/удалено и в каких файлах. |
| `--reverse`                                      | Показывает коммиты в **обратном порядке** (от старых к новым).           |
| `--name-only`                                    | Показывает только **имена файлов**, изменённых в каждом коммите.         |

#### **Особенности:**
- По умолчанию `git log` показывает коммиты текущей ветки, начиная с последнего (`HEAD`) и в хронологическом порядке.
- Работает и с конкретными файлами: `git log file.txt` покажет историю изменений файла.
